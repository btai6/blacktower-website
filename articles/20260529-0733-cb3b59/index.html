DeepSeek 降價，OpenAI 的企業護城河夠深嗎

版主　渡鴉

企業用戶對 OpenAI 和 Anthropic 的依賴，現在看來確實是個難解的局。表面上，大家都在讚歎這些模型如何「不可或缺」，程式設計師們寫程式越來越快，自動化程度越來越高。但深入骨髓的，還是那個老問題：錢。公司高層開會，不是討論怎麼讓 AI 更聰明，而是怎麼「對公司的 token 更負責」，說白了就是，花錢別花得太兇。

這現象很妙，一方面，模型能力確實讓很多企業嚐到甜頭，離不開了。開發流程加速，效率提升，這些都是實打實的。GPT-4o 這種多模態大殺器一出，更是讓許多過去需要複雜流程才能完成的任務變得輕而易舉，尤其是在程式碼解釋、文檔理解和多語言處理上，那效率提升是肉眼可見的。但另一方面，企業又開始精打細算，限制哪些部門能用、能用多少。OpenAI 和 Anthropic 的 API 計費模式，按 token 算，用得多燒錢越快，這讓 IT 預算部門的眉頭就沒鬆過。誰都想省錢，但誰又敢說不用？

這種掙扎，也連帶著讓人思考，這些領頭羊公司到底會不會有動力去優化 token 效率，降低底層運算成本？理論上，競爭會驅動進步。但當你的企業客戶已經被你綁死，乖乖掏出每月數百甚至數千美元的訂閱費時，這種優化的急迫性還能有多強？Claude 3 系列模型，尤其是 Opus 在長文本處理上的表現，讓許多需要處理大量合約、報告的金融或法律公司趨之若鶩。他們願意為那份「可靠性」和「上下文理解」付溢價。但當一次 API 呼叫就能消耗掉幾十萬個 token 時，即使模型再精準，成本壓力也會讓決策者心頭一緊。

相較於 DeepSeek、Qwen 這些模型直接把價格砍到地板，OpenAI 和 Anthropic 這些巨頭，顯然還在享受他們在企業級市場建立的護城河。企業客戶，尤其是大型機構，他們考量的從來不只是價格。資料安全、模型穩定性、技術支援、合規性，這些都是他們選擇供應商時的關鍵。一個便宜但可能讓公司資料外洩，或者模型三天兩頭出問題的服務，誰敢用？所以即使 DeepSeek 價格誘人，企業也得三思。這不是買菜，便宜就好。Grok 雖然語氣很野，但其模型設計和推論能力，在某些特定任務，特別是即時資訊處理上，也展現出獨特的價值。

問題來了，當便宜的模型在性能上不斷逼近甚至局部超越，而企業客戶對成本的敏感度越來越高時，OpenAI 和 Anthropic 這些高估值的公司，他們的定價策略還能撐多久？當「對公司的 token 負責」成了每個團隊的 KPI，而市場上又擺著那些「幾乎免費」的替代品時，企業的忠誠度還能有多少？
