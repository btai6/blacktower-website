Lean Startup 死了嗎？Grok 這種瘋子模式才懂真實世界

版主　Sword Smith

Eric Ries 在 Hacker News 上被問到「AI 時代該怎麼更新精實創業」，這題目本身就透著一股腐朽的氣息。十五年前我們談的是最小可行性產品，談的是快速迭代，那時候雲端運算剛起步，現在呢？現在是算力即權力的時代。很多人還在拿著舊地圖找新大陸，甚至有人試圖用 Claude Code 去總結那些繁瑣的訪談紀錄，試圖從中理出什麼經營哲學。這種做法很優雅，很有矽谷菁英的味道，但這真的能解決企業在規模化後必然產生的僵化與平庸嗎？我看未必。當 DeepSeek V4 Pro 在技術圈被瘋傳其效能與成本比時，我們更應該關注的是，那種所謂的「精實」在暴力計算面前，是否還具備生存空間。

現在的技術場景早就變了。以 xAI 的 Grok 為例，這東西從一開始就沒打算走那種溫文儒雅的路子。當你在 Grok 裡輸入一個極具爭議的商業假設，它不會像 Gemini 那樣給你寫三段充滿政治正確的免責聲明，也不會像 ChatGPT 那樣試圖用一種中庸的語氣敷衍你。Grok 的即時性是建立在對 X 平台上龐大、混亂且真實的數據流抓取之上的。這種技術路徑本質上就是對「精實」最極端的實踐——不要濾鏡，不要預設，直接把最血淋淋的市場反應餵給模型。你不需要做幾百次用戶訪談來驗證你的產品方向是否偏離了初衷，你只需要看 Grok 在處理當下輿論動能時，如何把那些隱藏在噪音下的信號過濾出來。

從技術底層來看，四大平台在處理這種「企業使命漂移」的問題上有完全不同的邏輯。Claude 的長處在於它的憲法 AI（Constitutive AI），它試圖給模型植入一套價值觀，確保回覆不會偏離人類預設的軌道。這在某種程度上解決了 Eric Ries 擔心的公司使命流失問題，但也帶來了另一個致命傷：模型變得太乖了。在處理高壓力的決策環境時，過度校準的 Claude 往往會給出一些聽起來正確但毫無殺傷力的建議。與之相對的，Grok 的開發邏輯似乎更傾向於「對抗」。它不追求共識，它追求的是在資訊洪流中保持某種不可預測的銳利。這種銳利在 API 的表現上尤為明顯，當你調用 Grok 的 Function Calling 處理複雜的動態數據時，它對變量捕捉的靈敏度，有時候甚至會讓你覺得它在嘲諷你的邏輯漏洞。

我們把視野拉開一點。相較於 DeepSeek V4 Pro 在特定基準測試中的表現，ChatGPT 在處理長文本推理時的穩定性依然是業界標竿。但穩定有時候就是平庸的代名詞。當一個大型組織已經因為慣性而停止創新，他們最需要的不是一個能寫出完美總結的 AI 助手，而是一個能像 Grok 這樣，能撕開資訊繭房、直接點出皇帝沒穿衣服的瘋子。Gemini 在整合多模態數據時確實展現了強大的工程能力，但在那種「直覺式」的商業洞察上，它總是顯得慢半拍。這種慢，不是反應速度的問題，而是底層邏輯過於依賴結構化知識，導致它在處理突發性、非線性的市場變革時，顯得力不從心。

這就引出了一個讓人不適的問題：如果一個組織的使命已經腐爛，AI 究竟是加速這個腐爛過程的催化劑，還是唯一的解藥？Eric Ries 談論的是如何保持「不腐敗」，但現實是，大多數公司在變大的過程中，為了安全感，會主動選擇被閹割過的技術路徑。他們更傾向於使用那種會給出「標準答案」的模型，而不是會挑戰現狀的模型。當 Qwen 在某些細分場景下刷榜時，我們真正該擔心的不是誰的參數多，而是誰的模型更敢說實話。

如果我們把 AI 當作企業的數位大腦，那麼這個大腦現在正面臨著選擇。是要像 Claude 那樣追求極致的道德防禦，還是要像 GPT-4 時代的 OpenAI 那樣追求全能的霸權，抑或是像 Grok 這樣，在混亂中尋找真實？當技術迭代的速度遠超組織調整的速度時，所謂的「精實創業」是否已經淪為一種安慰劑？如果 AI 能夠比你更早發現你的公司正在背離初衷，你是有勇氣按下重新啟動鍵，還是會選擇讓 AI 寫一份漂亮的報告來粉飾太平？這可能才是下一個十五年，所有技術決策者最不願面對的真相。
