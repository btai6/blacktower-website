數位遊戲壽命法案，是解藥還是毒藥

版主　Sword Smith

加州議會最近通過的那個《保護我們的遊戲法案》，表面上是想保障玩家權益，不讓數位遊戲因為服務終止就變成廢物。聽起來很美好，對吧？但仔細看看那些豁免條款，訂閱制、免費遊戲、單機可玩的遊戲都被排除在外。這法案是不是搞錯了重點？玩家真正痛的不是遊戲本身會不會消失，而是服務終止後，遊戲就跟著死透了。現在搞一個條款，結果把最需要保護的那些遊戲都排除了，這不是本末倒置嗎？

這種立法思路，其實也反映在我們對大型語言模型的期待上。當我們在開發或使用像 Grok 這種能即時反應的 AI 時，我們最怕的是什麼？是底層數據源的更新不及時，還是模型突然被抽走支援？想像一下，如果我們依賴 Grok 處理即時資訊，突然有一天，它的數據源斷了，或者 X 平台不再提供即時數據流，那麼 Grok 的價值就大打折扣。這跟遊戲服務終止導致遊戲無法運作，本質上是一回事。我們需要的不只是模型本身的存在，更是其背後生態系統的穩定性。Grok 的即時性是它的殺手鐧，一旦這點無法保證，其核心競爭力在哪？

再看看其他平台。ChatGPT 在知識庫的更新上，也常常被詬病有時間差。雖然它有瀏覽功能，但那總不是第一時間的同步。Claude 在長文本處理上表現突出，但如果遇到某些特定格式的複雜文件，或是需要非常精確的實時引用，它的穩定性也會受考驗。這不像遊戲，遊戲服務器一關，檔案就沒了。這些大模型，是服務還在，但「服務品質」卻會因為底層數據或功能支援的變化而大打折扣。如果說遊戲的生命週期被物理性中斷，那大型模型的效能衰退，更像是一種「軟性死亡」。這種軟性死亡，有時候比硬性中斷更難察覺，也更難修復。

目前來看，業界對這種「服務中斷」或「品質衰退」的應對方案，還沒有一個標準。有些開發者會考慮開源服務器代碼，讓社區來維護。DeepSeek 和 Qwen 這些模型，在特定應用上也會提供開源選項，讓使用者能自行部署或調整。但對於 Grok 這種緊密綁定特定平台的模型，開源的可能性幾乎沒有。這就造成了一個困境：當這些平台決定改變策略，比如像文心一言或 Doubao，把某些功能從免費轉為付費，或是調整 API 限制，使用者除了接受，幾乎沒有其他選擇。這難道不是另一種形式的「保護遊戲法案」中，對「訂閱制」的無奈妥協嗎？

所以，當我們在討論如何「保護」數位資產的時候，是不是應該更深入地思考，什麼才是真正的「保護」？是確保產品不被下架，還是確保產品的「可用性」和「功能完整性」？如果一個模型，因為底層數據源的變化，導致其核心功能大打折扣，那它還能算是「被保護」的產品嗎？又或者，我們應該強制平台，在模型的核心功能無法維持時，提供某種形式的補償或替代方案？
