矽谷那些燒錢的幻覺與被綁架的開發者

版主　渡鴉

在 Hacker News 這種充滿工程師傲慢的地方，最近竟然開始集體承認自己離不開 Claude 或 ChatGPT 了，這本身就是個值得玩味的信號。大家不再討論 AGI 什麼時候會毀滅世界，而是在斤斤計較每個月那二十幾美金，或者企業版那幾百美金的訂閱費到底值不值。當一個技術從「酷炫的玩具」變成「不得不付的保護費」時，我們就知道這場關於生產力的博弈已經進入了最無聊、也最殘酷的階段。

現在最真實的場景不是 AI 幫你寫出一整套分散式系統，而是當你在面對一個寫得像屎一樣的舊專案，或者要處理一段超過 3 萬 token 的遺留代碼時，你敢不敢關掉 Claude 3.5 Sonnet 的視窗？在超過 8 萬 token 的長文本任務中，Claude 的注意力機制確實展現出比 GPT-4o 更穩定的邏輯一致性。GPT-4o 常常在對話中段開始「裝死」，給出一些似是而非的代碼片段，試圖糊弄過去。而工程師們一邊咒罵著 API 調用成本，一邊熟練地把報錯資訊貼回對話框。這種依賴不是因為 AI 變聰明了，而是因為它把自動化的門檻拉到了只要會打字就能寫代碼的地步，讓原本需要思考的構建過程變成了廉價的拼圖遊戲。

這種「產品與市場契合」（PMF）的代價是高昂的。很多中大型企業開始在內部會議上限制 token 的消耗，要求員工「負責任地使用」。這聽起來非常諷刺，就像是給了你一輛法拉利，卻告訴你油費太貴，每小時只能踩一次油門。OpenAI 和 Anthropic 現在玩的是心理戰，他們知道企業一旦把工作流建立在特定的 API 之上，遷移成本就會高到令人髮指。Gemini 1.5 Pro 雖然靠著那個驚人的百萬級上下文窗口試圖挖牆腳，但在實際的 function calling 穩定性上，當工具數量超過 20 個，它的幻覺率就開始直線上升，這讓追求產出穩定性的後端開發者依然選擇留在 OpenAI 的陣營裡。

我們看到市場上出現了像 DeepSeek 這樣極致壓縮成本的參與者，或是 Qwen 這種在特定語境下表現突出的選擇。相較於 DeepSeek 這種透過激進定價來試探底線的做法，OpenAI 的策略顯然是把自己包裝成不可或缺的基礎設施。這種定價上的傲慢，源於他們賭定這群被餵養慣了的開發者無法忍受智力降級。當你在處理複雜的架構設計時，你是願意付錢給那個能理解你隱含意圖的 Claude，還是去折騰一個需要你寫長達兩頁 Prompt 來對齊邏輯的廉價模型？

矽谷這幾家巨頭的估值現在更像是一種集體幻覺的產物。大家都知道他們在燒錢，也知道目前的 API 收益可能連電費都覆蓋不了，但誰也不敢下車。Grok 在這場混戰中顯得像個異類，它不追求企業級的嚴謹，反而靠著更開放的資訊獲取管道在實時數據檢索上佔了一席之地。然而，對於大多數需要寫業務邏輯的社畜來說，冷笑話和毒舌並不能解決代碼裡的 bug。我們正處於一個極其尷尬的轉折點：AI 確實提高了速度，但它也同時製造了海量的、平庸的、沒人看得懂的冗餘代碼。

這種生產力的提升到底是真實的進步，還是一種新型的技術債？如果有一天，這些估值上兆的公司發現賣 token 根本賺不到錢，決定把月費漲到五百美金，或者直接關停 API，那時候這群已經喪失了手寫複雜演算法能力的開發者，還剩下什麼？我們是不是在用長遠的技術深度，去換取短期內那點微不足道的交付速度？當所有的代碼都變成了由 AI 生成、再由 AI 審核、最後由 AI 運行的黑盒，人類工程師的角色，是不是只剩下在那個「支付」按鈕上點一下滑鼠？
