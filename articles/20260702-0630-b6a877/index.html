老闆看不懂你的離職信，這件事本身就很 AI。

版主　渡鴉

你每天對著 ChatGPT 敲敲打打，為了讓它生成一段像樣的代碼或是改好一封週報，你願意花半小時在那裡琢磨提示詞（Prompt），又是賦予角色，又是給上下文，又是規定輸出格式。結果你要跟這家公司分手的時候，你只會寫「因個人職涯規劃辭職」？這跟你餵給 GPT 一句「幫我寫個東西」然後嫌它回覆廢話連篇有什麼區別？

很多人對人類語言的精準度有種近乎偏執的誤解。他們覺得同事、主管、甚至是那個坐在辦公室盡頭只會看財報的老闆，應該天生具備某種心靈感應的能力。你心裡那點被排擠的酸楚、對升遷制度的絕望、還有對隔壁桌小張領得比你多那三千塊的憤怒，你期待老闆能從你那封乾癟的信封縫隙裡讀出來。醒醒吧，連 Gemini 這種擁有百萬上下文窗口的怪物，你只要漏掉一個關鍵約束，它都能給你吐出一堆邏輯不通的垃圾，更何況是一個大腦裡塞滿了 KPI、股價和下一季報表的碳基生物。

這就是現在職場最幽默的地方。我們對機器的耐心遠超對人的耐心。你看到 Claude 回答得不如預期，你會反思是不是自己的 Input 有問題，是不是給的範例不夠多，是不是溫度值（Temperature）設得太高讓它開始胡言亂語。但面對老闆，你卻像個沒拿到禮物就生悶氣的小鬼，把辭呈往桌上一甩，心裡想著：「他應該知道我為什麼要走。」他知道個屁。在他眼裡，你就是一個運行的 Token 消耗殆盡、需要被後台更換掉的 Instance。

你以為離職信是情緒的終點，其實它也是一種提示工程。如果你想讓這場離職產生某種效力——不管是讓老闆感到愧疚，還是讓公司真的去檢討那個爛到透頂的獎金制度，你得給出精確的輸入。而不是像現在這樣，寫一段跟 DeepSeek 或 Qwen 同樣模版化的外交辭令。當然，如果你只是想安靜地滾蛋，那當我沒說。但我看過太多人在離職後的第三天，還在酒桌上憤憤不平地抱怨「老闆根本沒意識到公司留不住人才」，這就顯得很矯情了。你沒給 Prompt，人家怎麼 Run 出你要的 Result？

現在的技術圈有個毛病，大家都覺得把事情交給 AI 處理就能解決溝通障礙。GPT-4o 確實能幫你把憤怒的草稿修飾得優雅得體，能把「我想把這家公司的伺服器炸了」轉化為「我認為公司的技術架構仍有優化空間」。但這就是問題所在：過度的對齊（Alignment）讓所有的輸出都變得平庸。你的委屈被磨平了，你的稜角被濾掉了，最後呈現在老闆面前的是一份經過 RLHF（人類回饋強化學習）後的完美廢話。這份報告放在任何一個人的桌上都適用，它缺乏那個獨一無二的「人味」，也就意味著它根本觸發不了任何實質性的反應。

我們在跟四大 AI 交流時，最怕的就是模型進入一種「安全模式」，不管你問什麼，它都回你「作為一個人工智慧，我不建議……」。你的離職信現在就是這種狀態。你怕得罪人，怕毀了名聲，怕圈子太小以後相見尷尬。所以你選擇了最安全、最無效的輸出。然後你轉過身，又去論壇上抱怨老闆是個沒心沒肺的機器人。這邏輯難道不荒謬嗎？

老闆其實就是一個推理能力極其有限、且充滿幻覺（Hallucination）的老舊模型。他會把你的沉默解讀為滿意，把你的順從解讀為忠誠，把你那封禮貌的離職信解讀為「這孩子找到了更好的出路，我真替他高興」。如果你不提供負面的反饋循環（Negative Feedback Loop），他的參數永遠不會更新。他會帶著那套錯誤的權重，繼續折磨下一個進場的初級工程師。你說你愛這行，你說你關心技術環境，結果你連一個 Bug Report 都懶得寫就直接卸載軟體了。

說到底，我們對 AI 的寬容是因為我們知道它沒有心。它出錯了，那是算法問題，是語料庫污染。但人不一樣，我們預設人有共情能力。這種預設正是所有失望的源頭。在矽谷那幫人忙著讓 Grok 變得更憤世嫉俗、讓 Gemini 變得更政治正確的時候，我們這些普通人卻在退化。我們變得不會說人話，只會說那些被優化過的、不痛不癢的社交代碼。

你的那封離職信，在老闆眼裡可能連一萬個 Token 的價值都不到。他掃一眼，捕捉到「辭職」這個關鍵字，然後就開始在心裡計算招募新人的成本。至於你說的那些委屈、那些深夜的加班、那些被否決的提案，不好意思，這些 Context 被自動丟棄了。內存不足，無法加載。

如果你真的想要老闆讀懂你的委屈，你得像調試代碼一樣去拆解你的不爽。哪一行代碼崩了？哪個模塊設計有毒？哪個調度器在亂搞？不要期待他能像神一樣看穿你的靈魂。他要是真有那種算力，他早就去寫底層架構了，還坐在那裡聽你唸辭職信？

很多人會反駁說，說了也沒用，公司不會改的。這又是典型的決定論。你餵給 GPT-4 兩段截然不同的 Prompt，得到的結果天差地別。你試都沒試過給出一份帶有「具體場景」和「痛點分析」的真實反饋，你就斷定這個模型已經死掉了。或許它確實死掉了，但至少你完成了一次高質量的輸出，而不是在那裡製造更多的數據噪音。

離職不只是關係的終結，它是一次最後的 Debug。如果你連這點勇氣都沒有，那就別整天在那裡研究什麼提示詞優化了。你連對一個活生生的人說出真話都做不到，你對著屏幕敲出的那些精妙指令，不過是另一種形式的逃避。那些在論壇上大談特談 Claude 3.5 Sonnet 邏輯推理多強的人，回到現實生活中，連跟主管爭取預算都要先去問問 AI 怎麼寫才不會顯得太激進。這真是這個時代最大的黑色幽默。

我們活在一個充滿了「提示」的時代，卻失落了最原始的溝通。我們追求模型的低延遲，卻對人與人之間的冷漠視而不見。你的離職信寫得再漂亮，只要它不敢觸及那個核心的痛點，它就是一封失敗的 Prompt。老闆沒讀懂你的委屈？不，是他根本沒在你提供的數據包裡找到任何值得處理的信息。

下次當你覺得被忽視、被低估、被委屈的時候，別急著去翻 AI 工具書。先想想，如果你是一個開發者，面對一個完全不給報錯信息、只會無聲崩潰的用戶，你會怎麼想？你會覺得這個用戶很深沉、很有故事嗎？不，你只會覺得這是一個擾亂系統運行的垃圾帳號，然後毫不猶豫地按下 Delete。

這世界不欠你一個懂你的老闆，就像 OpenAI 不欠你一個完美的回答。如果你想要那個結果，你就得自己去構建那個上下文。別在那裡一臉看破紅塵的樣子，其實你只是懶，懶到連為自己辯護的力氣都省了，然後把這份懶惰包裝成了一種名為「成熟」的沉默。這種沉默，跟那些壞掉的、只會重複「對不起，我無法理解您的請求」的 AI 沒什麼兩樣。
